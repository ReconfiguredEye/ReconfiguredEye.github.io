Recoding Schotter

##### Prompt Schotter by Georg Nees (1968) is a classic algorithmic artwork that depicts a gradient from order to chaos across a 12x22 grid of squares. In it, the squares' orientations become increasingly randomized towards the bottom of the page. Recode this work. ##### Response In the artwork, the squares gradually become more and more randomized as you move downwards, both in their orientation (rotation) and in their positioning (translation). Here's how you can recreate it: ```javascript let gridX = 12; // Number of squares in the X direction let gridY = 22; // Number of squares in the Y direction let squareSize = 40; // Size of the squares let chaosFactor = 0.05; // Factor to control the amount of chaos (increase for more chaos) function setup() { createCanvas(800, 800); noLoop(); } function draw() { background(220); translate(width / 2 - gridX * squareSize / 2, height / 2 - gridY * squareSize / 2); // Center the grid for(let i = 0; i < gridX; i++) { for(let j = 0; j < gridY; j++) { push(); // Save the current transformation matrix translate((i + 0.5) * squareSize, (j + 0.5) * squareSize); // Move to the center of the current square let chaosLevel = j / gridY; // Compute the level of chaos based on the current row // Add some random translation and rotation to the square, scaled by the chaos level translate(random(-1, 1) * squareSize * chaosFactor * chaosLevel, random(-1, 1) * squareSize * chaosFactor * chaosLevel); rotate(random(-PI, PI) * chaosFactor * chaosLevel); square(-squareSize / 2, -squareSize / 2, squareSize); // Draw the square centered at the origin pop(); // Restore the transformation matrix } } } ``` In this script, `setup()` creates a canvas and sets it to not loop, and `draw()` draws a 12x22 grid of squares where each square's position and orientation is slightly randomized, with the amount of randomness (chaos) increasing as you move downwards. The transformation matrix is saved before drawing each square and restored afterwards, so the transformations do not accumulate. You can adjust the parameters as needed, such as the number of squares in the grid, the size of the squares, and the factor controlling the amount of chaos. The positioning of the grid can also be adjusted to center it on the canvas, if needed.
