vocabulary

1. **Sketch**: The term for a program or piece of code in p5.js. 2. **Canvas**: The space where you draw and animate in p5.js. 3. **Setup()**: A function in p5.js that is run once when the program starts. It's used to define initial environment properties. 4. **Draw()**: A function in p5.js that runs continuously after setup(). Used for animation purposes. 5. **Render**: The process of displaying a sketch or drawing on the screen. 6. **FrameRate**: The number of times the draw() function executes in one second. 7. **Shapes**: Basic building blocks in p5.js such as circles, squares, and triangles. 8. **Fill**: A function that sets the color used to fill shapes. 9. **Stroke**: A function that sets the color used to outline shapes. 10. **Coordinates**: An ordered pair of numbers defining a point on the canvas. The x-coordinate typically represents the horizontal axis and the y-coordinate represents the vertical axis. 11. **Variables**: Named symbols that can hold a value. In p5.js, variables are often used to store values like color, position, size, etc. 12. **Functions**: Blocks of code designed to perform a particular task. They can be built-in, such as setup() and draw(), or user-defined. 13. **Arrays**: A data structure that can store a fixed-size sequence of elements of the same type. 14. **Loops**: Code structures used to repeat a block of code as long as a specified condition is met. 15. **Objects**: Instances of classes, which encapsulate both data and functions that manipulate that data. 16. **Classes**: In object-oriented programming, classes define the properties and methods that will belong to objects. 17. **Methods**: Functions defined inside a class. 18. **Instances**: Objects created from a class, each with its own set of properties. 19. **Algorithm**: A step-by-step procedure for calculations, used for data processing and automated reasoning. 20. **Randomness**: Unpredictability. In p5.js, randomness is often used to make things appear more natural or less programmed. 21. **Perlin Noise**: A function developed by Ken Perlin that generates smooth, natural-looking randomness. 22. **API**: Stands for Application Programming Interface. It is a set of rules and protocols for building and interacting with software applications. 23. **Interactive Programming**: Programming that allows interaction between humans and computers, such as mouse clicks, key presses, etc. 24. **Event Handler**: A callback routine that operates asynchronously and handles inputs received into a program. 25. **Callback Function**: A function passed into another function as an argument, which is then invoked inside the outer function. 26. **Animation**: The rapid display of a sequence of static images or models creating an illusion of movement. 27. **Particle Systems**: A technique in game physics, motion graphics, and computer graphics that uses many minute particles to represent a fuzzy object.

development process

1. **Conceptualize and Ideate**: Creative coding starts with an idea or a concept. This might be driven by a desired visual outcome, an interaction, a data source, or some combination thereof. It's a good practice to sketch, list, or otherwise visually represent your idea to make sure you have a clear understanding of what you want to create. You might also research and explore similar projects, tools, techniques, and technologies at this stage. 2. **Plan**: Once you have a clear idea, planning becomes crucial. You'll want to break your idea down into manageable parts. If it's a complex project, you might want to plan out which libraries or tools you will need, and in what order you'll tackle the various components. If it's a simple project, you may just need to decide the sequence of functions or methods you will write. 3. **Design**: This phase often overlaps with the planning phase in creative coding. You will design the visual aspect, the interaction, the structure of the code, etc. You'll make decisions about colors, shapes, and layout, as well as how the user will interact with your creation. In the code structure design, you decide how to modularize the code and which parts of the code could be made into functions or objects. 4. **Implement**: This is the stage where you start coding. You'll write your functions, create your visuals, and start to see your idea take shape. You might use p5.js, Processing, openFrameworks, or any other creative coding tool. 5. **Test**: Testing in creative coding is just as important as in traditional software development. Does your creation work as expected? How does it respond to different user inputs? Is it visually appealing and engaging? If there are any bugs or glitches, this is the time to correct them. 6. **Iterate**: Rarely does the first version of a creative coding project turn out exactly as planned. Iteration, based on both testing and feedback, is a key part of the creative coding process. You'll make changes, experiment with different approaches, and refine your project. 7. **Presentation**: In creative coding, how you present your work is vital. This could be in an online portfolio, a physical installation, or a video demonstration. You might also need to consider documentation - explaining how your project works and what it means. 8. **Review and Reflect**: After your project is complete and presented, it's important to reflect on the process. What did you learn? What would you do differently next time? How could this project inform your future work? Reflection is an important part of growth as a creative coder.
